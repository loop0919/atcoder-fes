# AtCoder フェス Discord Bot 要件定義

## 利用者

| 利用者 | 説明 | 主な関心 |
|---|---|---|
| 参加者 | フェスに参加する Discord サーバーメンバー。AtCoder アカウントを登録し、陣営に投票してコンテストに参加する。 | 簡単に参加登録できること、自分の陣営・スコア・順位を確認できること |
| フェス管理者 | フェスの作成、テーマ設定、対象コンテスト設定、評価関数設定、結果確定を行う運営者。 | イベント進行を Discord 上で完結できること、公平な集計・調整ができること |
| Discord サーバー管理者 | Bot の導入、権限付与、利用可能チャンネルや管理ロールを設定する人。 | Bot 権限を最小限に保てること、管理対象を制御できること |
| 観戦者 | フェスには参加しないが、途中経過や最終結果を見るメンバー。 | 進行状況や結果が分かりやすく共有されること |
| Bot 運用者 | Bot のデプロイ、ログ監視、障害対応、設定変更を行う人。 | 安定稼働すること、障害原因を追えること、秘密情報が安全に管理されること |

## スコープ

### 対象範囲

- Discord 上でフェスを作成・管理できる。
- フェスごとにお題と選択肢を設定できる。
  - 例: `ペットにしたいのはどっち？`
  - 選択肢例: `ねこ`, `くらげ`
- 参加者は Discord Bot を通じて、どちらの陣営に所属するか投票できる。
- 参加者は Discord アカウントに AtCoder ID を紐付けできる。
- なりすまし防止のため、Bot が発行した認証トークンを AtCoder プロフィールの所属欄に入力させ、Bot が一致確認する。
- フェス管理者は対象となる AtCoder コンテストを設定できる。
- Bot は対象コンテストの成績情報を取得し、参加者ごとの成績を集計できる。
- Bot はチーム別のスコアを算出できる。
- フェス管理者は、現在レート・コンテストパフォーマンス・順位などを使った評価関数を設定できる。
- 評価関数は、集計前にサンプルデータまたは実データでプレビューできる。
- Bot は最終結果を Discord チャンネルに投稿できる。
- 管理者操作、認証失敗、集計失敗などの重要イベントをログとして残せる。

### 対象外

- AtCoder のコンテスト自体を作成・開催する機能。
- AtCoder アカウントへのログイン代行。
- AtCoder の問題提出、解説、バーチャル参加の自動操作。
- Discord サーバー外の一般公開ランキングサイト。
- 決済、賞品発送、個人情報管理を伴う機能。
- 初期リリース時点での専用 Web 管理画面。
- 管理者が任意のプログラムコードを無制限に実行できる仕組み。

### 前提・制約

- Bot は Discord のスラッシュコマンドを主な操作インターフェースとする。
- フェス管理者向け操作は、Discord ロールまたは権限によって制限する。
- AtCoder ID は Discord ユーザーごとに一意に登録される。
- 1つの AtCoder ID を複数の Discord ユーザーが同時に登録することはできない。
- 認証トークンは十分に推測困難な文字列として生成する。
- 評価関数は安全に実行できる形式で定義する。
  - 例: 式 DSL、設定式、またはサンドボックス化された実行環境。
- 集計結果は、管理者による確認後に確定・公開できる。

## ユースケース

| ID | 利用者 | ユースケース | 概要 | 成功条件 |
|---|---|---|---|---|
| UC-001 | 参加者 | AtCoder ID を登録する | 参加者が Discord 上で AtCoder ID を入力し、Bot が認証トークンを発行する。 | Discord ユーザーと AtCoder ID の仮登録が作成される |
| UC-002 | 参加者 | AtCoder ID を認証する | 参加者が AtCoder の所属欄に認証トークンを入力し、Bot に確認を依頼する。 | Bot が所属欄を確認し、トークン一致時に登録済み状態へ変更する |
| UC-003 | 参加者 | フェスに参加する | 参加者が開催中または参加受付中のフェスに参加表明する。 | 参加者がフェスの参加者一覧に追加される |
| UC-004 | 参加者 | 陣営に投票する | 参加者がお題に対して、いずれかの選択肢を選ぶ。 | 参加者の陣営が記録される |
| UC-005 | 参加者 | 自分の登録状況を確認する | 参加者が AtCoder 認証状態、参加中フェス、選択済み陣営を確認する。 | Bot が現在の状態を本人に表示する |
| UC-006 | フェス管理者 | フェスを作成する | 管理者がフェス名、お題、選択肢、対象コンテスト、受付期間を設定する。 | 新しいフェスが作成され、参加受付可能になる |
| UC-007 | フェス管理者 | 投票・参加受付を締め切る | 管理者がフェスの受付を締め切り、陣営を確定する。 | 以後、参加者は陣営を変更できない |
| UC-008 | フェス管理者 | 評価関数を設定する | 管理者がコンテスト成績から参加者スコアを算出する評価関数を設定する。 | 評価関数が保存され、プレビュー可能になる |
| UC-009 | フェス管理者 | 評価関数をプレビューする | 管理者がサンプルデータまたは取得済み成績を使って、スコア計算結果を確認する。 | 参加者別・陣営別の試算結果が表示される |
| UC-010 | フェス管理者 | コンテスト結果を集計する | 管理者が対象コンテスト終了後に成績取得とスコア計算を実行する。 | 参加者ごとのスコアと陣営別スコアが算出される |
| UC-011 | フェス管理者 | 結果を公開する | 管理者が集計結果を確認し、Discord チャンネルに公開する。 | 最終結果、勝利陣営、主要な個人成績が投稿される |
| UC-012 | 観戦者 | フェスの状況を見る | 観戦者が現在の参加人数、陣営分布、結果を確認する。 | Bot が公開可能な範囲のフェス情報を表示する |
| UC-013 | Bot 運用者 | 障害や失敗を確認する | 運用者が認証失敗、AtCoder 取得失敗、集計失敗などを確認する。 | ログから失敗原因と対象フェスを追跡できる |

## 機能要件

優先度は以下の基準で定義する。

- Must: 初期リリースに必須。
- Should: 初期リリースに含めたいが、代替運用が可能。
- Could: 余裕があれば対応する。
- Won't: 現時点では対応しない。

### アカウント登録・認証

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-001 | AtCoder ID 登録 | 参加者は Discord コマンドで AtCoder ID を登録できる。 | Must |
| F-002 | 認証トークン発行 | Bot は AtCoder ID 登録時に、ユーザーごとに推測困難な認証トークンを発行する。 | Must |
| F-003 | 所属欄による本人確認 | Bot は AtCoder プロフィールの所属欄を取得し、認証トークンと一致するか確認できる。 | Must |
| F-004 | 認証状態管理 | Bot は AtCoder ID の状態を `未認証`, `認証済み`, `認証失敗` などで管理できる。 | Must |
| F-005 | 重複登録防止 | 同一 AtCoder ID を複数の Discord ユーザーが同時に認証済みにできない。 | Must |
| F-006 | 登録情報確認 | 参加者は自分の AtCoder ID と認証状態を確認できる。 | Must |
| F-007 | 登録解除・再登録 | 参加者または管理者は、必要に応じて AtCoder ID の登録解除や再登録ができる。 | Should |

### フェス管理

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-101 | フェス作成 | フェス管理者はフェス名、お題、選択肢、説明、受付期間を設定してフェスを作成できる。 | Must |
| F-102 | 選択肢設定 | フェス管理者は2つ以上の選択肢を設定できる。初期リリースでは2択を標準とする。 | Must |
| F-103 | 対象コンテスト設定 | フェス管理者は AtCoder の対象コンテスト ID または URL を設定できる。 | Must |
| F-104 | フェス状態管理 | Bot はフェス状態を `下書き`, `受付中`, `受付締切`, `集計中`, `結果確定`, `公開済み` などで管理できる。 | Must |
| F-105 | 受付締切 | フェス管理者は参加受付および投票受付を締め切れる。 | Must |
| F-106 | フェス一覧表示 | Bot は現在受付中、開催中、終了済みのフェス一覧を表示できる。 | Should |
| F-107 | フェス詳細表示 | Bot はお題、選択肢、対象コンテスト、受付状況、参加人数を表示できる。 | Should |

### 投票・チーム分け

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-201 | 参加表明 | 認証済みの参加者はフェスへの参加表明ができる。 | Must |
| F-202 | 陣営投票 | 参加者はフェスのお題に対して、いずれかの選択肢へ投票できる。 | Must |
| F-203 | 投票変更 | 受付期間中であれば、参加者は投票先を変更できる。 | Should |
| F-204 | 締切後ロック | 受付締切後は、参加者が投票先を変更できない。 | Must |
| F-205 | 陣営人数表示 | Bot は各陣営の参加人数を表示できる。 | Should |
| F-206 | 未投票者確認 | フェス管理者は参加済みだが投票していない参加者を確認できる。 | Should |

### AtCoder コンテスト成績取得

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-301 | 参加者成績取得 | Bot は対象コンテストにおける参加者の順位、得点、パフォーマンス、レート変動などを取得できる。 | Must |
| F-302 | 現在レート取得 | Bot は参加者の現在レートまたはコンテスト前レートを取得・保持できる。 | Must |
| F-303 | 未参加者検出 | Bot はフェス参加者のうち、対象コンテストに参加していないユーザーを判定できる。 | Must |
| F-304 | 取得失敗時の再実行 | AtCoder からの取得に失敗した場合、管理者が再取得を実行できる。 | Should |
| F-305 | 取得データ確認 | フェス管理者は集計前に取得済みの参加者データを確認できる。 | Should |

### 評価関数・スコア計算

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-401 | 評価関数設定 | フェス管理者は、参加者のコンテスト成績から個人スコアを算出する評価関数を設定できる。 | Must |
| F-402 | 利用可能変数 | 評価関数では、AtCoder ID、陣営、現在レート、コンテスト前レート、パフォーマンス、順位、得点、参加有無などを参照できる。 | Must |
| F-403 | 上振れ指標対応 | 評価関数は、パフォーマンスと現在レートの差分など、レートに対する上振れ加減を扱える。 | Must |
| F-404 | 評価関数プレビュー | フェス管理者は、評価関数を保存する前に計算結果をプレビューできる。 | Must |
| F-405 | 評価関数バリデーション | Bot は無効な式、未定義変数、危険な処理、計算不能な値を検出して保存を拒否できる。 | Must |
| F-406 | チームスコア集計 | Bot は個人スコアを陣営ごとに集計し、合計・平均・中央値などの集計値を算出できる。 | Must |
| F-407 | 未参加者の扱い設定 | フェス管理者は未参加者のスコアを0点、ペナルティ、集計対象外などから選べる。 | Should |
| F-408 | スコア計算履歴 | Bot は使用した評価関数、取得データ、計算結果を履歴として保存できる。 | Should |

### 結果公開

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-501 | 集計結果確認 | フェス管理者は公開前に、個人スコア、陣営別スコア、勝敗を確認できる。 | Must |
| F-502 | 結果確定 | フェス管理者は集計結果を確定できる。確定後は再集計しない限り結果が固定される。 | Must |
| F-503 | Discord 投稿 | Bot は指定チャンネルに最終結果を投稿できる。 | Must |
| F-504 | 結果表示 | Bot は過去フェスの結果をコマンドで表示できる。 | Should |
| F-505 | 個人ランキング表示 | Bot はフェス内の個人スコアランキングを表示できる。 | Should |

### 権限・監査

| ID | 機能 | 要件 | 優先度 |
|---|---|---|---|
| F-601 | 管理者権限制御 | フェス作成、締切、評価関数設定、集計、結果確定は管理者権限を持つユーザーのみ実行できる。 | Must |
| F-602 | チャンネル制御 | Bot は結果投稿先、管理操作チャンネル、ログ投稿先を設定できる。 | Should |
| F-603 | 操作ログ | Bot は重要な管理者操作を記録できる。 | Should |
| F-604 | エラーログ | Bot は AtCoder 取得失敗、認証失敗、集計失敗などをログに残せる。 | Must |
| F-605 | 秘密情報管理 | Bot Token、DB 接続情報などの秘密情報をソースコードに含めず、環境変数などで管理する。 | Must |
